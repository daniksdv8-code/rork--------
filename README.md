# ПаркМенеджер — Система управления парковкой

## Быстрый старт

### Требования

- **Node.js** 18+ ([nvm](https://github.com/nvm-sh/nvm))
- **Bun** ([bun.sh](https://bun.sh/docs/installation))

### Установка

```bash
git clone <YOUR_GIT_URL>
cd <YOUR_PROJECT_NAME>
bun install
```

### Переменные окружения

Создайте файл `.env` в корне проекта:

```env
EXPO_PUBLIC_RORK_API_BASE_URL=https://your-server.com
```

Для локальной разработки через Rork-платформу переменные устанавливаются автоматически.

### Запуск

```bash
# Веб-превью (рекомендуется для первой проверки)
bun run start-web

# Мобильное превью (Expo Go)
bun run start
```

## Архитектура

### Backend (Hono + tRPC)

- Точка входа: `backend/hono.ts`
- API: `backend/trpc/routes/parking.ts`
- Все данные хранятся в файле `.data/parking-store.json` (создаётся автоматически)
- При перезапуске сервера данные **не теряются**

### Frontend (React Native + Expo Router)

- Табы: Дашборд, Заезд, Клиенты, Парковка, Ещё
- Модальные окна: Выезд, Оплата долга, Оплата месяца, Новый клиент

### Синхронизация

- **Сервер — единственный источник истины**
- Клиенты опрашивают сервер каждые 2 секунды
- При конфликте версий данные сервера имеют приоритет
- Restore из бэкапа автоматически синхронизируется на все устройства (через epoch-механизм)

## Первый запуск

При первом запуске система автоматически создаёт администратора:

| Поле   | Значение |
|--------|----------|
| Логин  | `admin`  |
| Пароль | `admin`  |

**Обязательно смените пароль** после первого входа:
Настройки → Мой профиль → Новый пароль → Сохранить

---

## Роли

| Роль        | Возможности |
|-------------|-------------|
| **Админ**   | Полный доступ: тарифы, пользователи, бэкапы, сброс данных, все операции |
| **Менеджер**| Заезд/выезд, приём оплаты, работа в рамках своей смены |

Создание менеджеров: Настройки → Добавить менеджера

---

## Тарифы

Тарифы задаются как **дневные ставки**:

| Тип | Нал (₽/день) | Безнал (₽/день) |
|-----|---------------|------------------|
| Месяц | 150 | 160 |
| Разово | 150 | 200 |

Сумма за месяц = дневная ставка × количество дней в календарном месяце.

Тарифы можно изменить: Настройки → Тарифы.

## Бэкап и восстановление

### Создание бэкапа

Настройки → Резервное копирование → Скачать бэкап

Файл `parking_backup_YYYY-MM-DD_HH-MM.json` сохраняется на устройство.

### Восстановление из бэкапа

Настройки → Восстановить из копии → Выбрать файл

После восстановления:
- Все данные на сервере заменяются данными из бэкапа
- Все клиенты (телефоны) автоматически получают обновлённые данные
- Приоритет всегда у восстановленных данных

---

## Безопасность

- Пароли хешируются (SHA-256 + salt) и **никогда** не отправляются клиентам
- При миграции со старых версий пароли автоматически хешируются при первом входе
- Данные пользователей (passwordHash, passwordSalt) не попадают в ответы API

---

## Хранение данных

Данные хранятся в файле `.data/parking-store.json`:

- Создаётся автоматически при первом запуске
- Атомарная запись (tmp-файл + rename) — защита от повреждения при сбое
- Папка `.data/` добавлена в `.gitignore`

При self-hosted деплое убедитесь, что у процесса сервера есть права на запись в директорию `.data/`.

## Self-hosted деплой

### Вариант 1: Bun на VPS

```bash
# На сервере
git clone <YOUR_GIT_URL>
cd <YOUR_PROJECT_NAME>
bun install

# Создайте .env с нужными переменными
# Запустите сервер (например, через pm2 или systemd)
bun run start
```

### Вариант 2: Docker (пример)

```dockerfile
FROM oven/bun:1
WORKDIR /app
COPY package.json bun.lock ./
RUN bun install --production
COPY . .
VOLUME /app/.data
EXPOSE 3000
CMD ["bun", "run", "start"]
```

```bash
docker run -d \
  -p 3000:3000 \
  -v parking-data:/app/.data \
  --name parkmanager \
  parkmanager:latest
```

Важно: монтируйте `.data/` как volume, чтобы данные сохранялись при пересоздании контейнера.
